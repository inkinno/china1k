천자문 플래시 카드 애플리케이션 기획 및 개발 지시서 (MVP)

본 문서는 개발 에이전트(AI Coder)가 비즈니스 로직, 데이터 모델, 보안 규격, UI/UX 구조를 오차 없이 이해하고 완벽한 단일 페이지 애플리케이션(SPA)을 구현할 수 있도록 작성된 기획 지시서입니다.

1. 프로젝트 개요 및 MVP 목표

서비스명: 천자문 마스터 (Chunja Flashcard)

컨셉: 게이미피케이션(포인트, 콤보, 배지) 요소를 결합한 천자문 플래시 카드 학습 및 평가 도구

MVP 핵심 범위: 1.  Google 인증 로그인 연동 및 개인화 세션 관리
2.  순차적(ID순) 및 수준별(획수 난이도순) 플래시 카드 학습 모듈
3.  오답 자동 연동형 퀴즈 테스트 모듈 (15문항 / 30문항 제공)
4.  틀린 한자만 반복 학습하고 조건부(2회 이상 연속 정답) 해제하는 오답노트 기능
5.  포인트를 매개로 한 상점 MVP(출석 수호권, 펫 꾸미기 예고) 구현
6.  안정적인 Firestore 데이터 동기화 알고리즘 (3분 쿨타임 수동저장 + 10분 주기 자동저장)

2. 레이아웃 및 UI 디자인 가이드

레이아웃 비율 (Percentage-based Layout):

사용자의 다양한 기기 환경(모바일, 태블릿, 데스크톱)에 맞춰 메인 컨테이너는 비율(%, vw, vh)을 기반으로 확장 및 축소되어야 하며 가로 스크롤을 절대 생성하지 않습니다.

단, 세부 컴포넌트의 가독성을 위한 내부 패딩, 마진, 둥근 모서리 등은 고정 픽셀(rem/px) 단위를 사용할 수 있습니다.

상단 네비게이션 바 (Sticky Header):

상단에 고정되어 항상 노출됩니다.

구성 요소: 서비스 로고(BI), 현재 학습 탭 컨트롤러(대시보드 / 진도 / 시험 / 오답노트 / 상점), 사용자 프로필 요약(획득 포인트, 연속 로그인 일수, 배지 카운트), 실시간 동기화 상태 표시 및 수동 [저장 & 로드] 버튼

3. 인증 및 데이터 저장 아키텍처 (Firestore 보안 정책)

개발 시 다음 3대 필수 연동 규칙을 엄격하게 준수해야 합니다.

[RULE 1] 엄격한 Firestore 경로 설계

공용 데이터 (Public Data): /artifacts/{appId}/public/data/{collectionName}

개인 학습 데이터 (Private Data): /artifacts/{appId}/users/{userId}/{collectionName}

적용 예시: 사용자의 학습 이력, 오답노트, 포인트 기록은 반드시 아래 경로에 저장합니다.

doc(db, 'artifacts', appId, 'users', userId, 'progress', 'status')

[RULE 2] 단순 쿼리 원칙 (No Complex Queries)

Firestore에서 다중 조건 필터링(where()), 정렬(orderBy()), 개수 제한(limit()) 조합을 절대 사용하지 마십시오.

간단한 doc 검색 혹은 전체 컬렉션을 가져온 뒤, 필터링 및 정렬은 JavaScript 메모리상에서 처리해야 색인 에러(Index Error)를 완벽히 예방할 수 있습니다.

[RULE 3] Auth 우선 실행 후 쿼리 제어

Firestore에 대한 어떠한 읽기/쓰기 명령도 사용자의 인증이 완료된 후에만 트리거되도록 안전장치를 구축하십시오.

인증 프로세스는 signInWithCustomToken을 최우선으로 시도하며 실패하거나 없을 시 signInAnonymously로 폴백합니다.

4. 데이터 저장 및 동기화 최적화 (Save / Load Logic)

성능 최적화와 서버 트래픽 낭비를 완화하기 위한 버퍼링 저장 알고리즘을 설계합니다.

매 액션마다 즉시 동기화하지 않음: 한자 카드 터치 혹은 퀴즈 정답 시 클라이언트 측 Local State만 업데이트합니다.

수동 [저장 & 로드] 제한: 상단 바의 수동 저장 버튼은 3분에 1회만 활성화됩니다. 클릭 시 쿨타임 타이머가 UI에 표시됩니다.

10분 주기 자동 저장: 지속적인 학습 도중 이탈을 막기 위해 10분 주기로 배경에서 백그라운드 자동 저장이 트리거됩니다.

데이터 변경점 비교 연산 (Dirty Checking):

저장 요청 시, 마지막으로 데이터베이스에서 로드한 데이터 객체의 JSON 문자열과 현재 상태 데이터의 JSON 문자열을 단순 비교합니다.

두 문자열이 완전히 동일한 경우 불필요한 네트워크 네트워크 트래픽을 아끼기 위해 Firestore setDoc/updateDoc 요청을 생략하고 저장 성공 토스트를 사용자에게 띄웁니다.

5. 비즈니스 로직 및 기능 명세

A. 획수 기준 난이도 테이블

1단계 (아주 쉬움): 1획 ~ 4획

2단계 (쉬움): 5획 ~ 8획

3단계 (보통): 9획 ~ 12획

4단계 (어려움): 13획 ~ 16획

5단계 (매우 어려움): 17획 이상

B. 재화 및 보상 시스템 (Point Economy)

1 포인트의 기준 가치: 학습자의 1분간 집중 노력에 해당하도록 밸런싱합니다.

포인트 획득 규칙:

새로운 플래시 카드 1개 첫 오픈 (암기 완료 클릭 시): +0.2 포인트

일반 퀴즈 1문제 정답: +0.5 포인트

오답노트 학습 퀴즈 1문제 정답: +1.0 포인트 (취약점 개선 특별 보상)

연속 정답 콤보 보상: 5콤보 달성 시 +2 포인트, 10콤보 달성 시 +5 포인트 일시 지급

상점 시스템 (MVP 단계):

연속 출석 보호권 (구매 가격: 30 포인트): 하루 미접속 시에도 연속 출석 기록(Streak)이 초기화되는 것을 방지하는 소모성 아이템.

펫 시스템 꾸미기 슬롯 (구매 가격: 100 포인트): 차후 개발 예정인 귀여운 한자 정령 펫의 데코레이션 아이템(모자, 안경 등)을 선구매 및 소유할 수 있는 컬렉션 메뉴 구현.

C. 평가(시험) 및 오답노트 알고리즘

퀴즈 대상 범위: 한 번이라도 플래시 카드를 열람하여 '학습 이력'이 남은 한자들로만 문제를 구성합니다.

문항수 선택: 15문제 또는 30문제를 시작 시점에 선택하도록 제공합니다.

오답노트 등록: 퀴즈 도중 오답이 발생한 즉시 해당 한자의 ID와 함께 wrongCount: 1 상태로 로컬 오답 리스트에 자동 저장됩니다. 이미 리스트에 존재한다면 카운트가 증가합니다.

오답노트 잠금 해제: 오답노트에 저장된 유니크한 틀린 한자의 총 개수가 최소 15개 이상 누적되었을 때만 '오답노트 탭'이 활성화되어 복습 시험에 진입할 수 있습니다. 15개 미만인 경우 진입을 제한하고 더 공부하라는 격려 다이얼로그를 표시합니다.

오답노트 졸업 조건 (연속 2회 성공): 오답노트 복습 평가 도중 특정 한자 문제를 연속으로 2회 이상 정답을 맞춘 경우, 비로소 오답 체크 상태가 완전 해제되며 오답노트에서 영구 제외됩니다.

시험 완료 후 즉시 저장: 일반적인 학습과 달리, 공식 시험(15문항/30문항) 및 오답노트 복습 평가가 끝난 시점에는 3분 쿨타임을 우회하여 예외적으로 즉시 데이터베이스에 변경 상태가 자동 동기화됩니다.

D. 대화형 모달 안내창 (Custom Inline Dialog)

브라우저의 불쾌한 기본 팝업(alert(), confirm())을 사용하지 마십시오.

모든 안내 및 경고(예: "저장 성공", "쿨타임 제한", "오답노트 진입 불가")는 현 화면 상단 레이어를 덮는 세련되고 차분한 반투명 커스텀 다이얼로그 오버레이로 통일하여 사용자 응답을 수집하도록 처리합니다.
